Introduction

This Imusify API guide provides a comprehensive listing of methods available to team or third-party developers to build powerful web and mobile music applications that deliver a streaming experience to end-users, buy or sell music online, manage personal profile and music

Please note that Imusify API works over the HTTP or HTTPS (later) protocol and currently supports JSON or JSON/HTML or HTML (chunk) response only.

Requests

Requests should be made up of the following components:

  • Route - to access an object, like Users or Tracks
  • Action - to perform a method related to the object, default Index
  • Parameters - to provide parameters required by the method (Form data, POST method)

To construct a proper request, you will need to format the URI as follow:

Live: http://www.imusify.com/imusify/{route}/{action}?{parameters} 

Development and Testing: http://beta.imusify.com/imusify/{route}/{action}?{parameters}

An example request to fetch About page and the response is in JSON 

Live: http://www.imusify.com/imusify/about/ 
POST method, Form data ajax=true

Development and Testing: http://beta.imusify.com/imusify/about/
POST method, Form data ajax=true

By default, requests are based on HTTP POST. However there are also methods to update objects which require GET data to be submitted.
Now there is no support for PUT or DELETE methods

Responses

The response of your request will contain the following:

  • HTTP status code (for example: 200 for success or 500 for error)
  • API response status if applicable (for example: ok or success for success or false for error)
  • API response status message (either a JSON object or array of strings or HTML or JSON with HTML chunk)

User/Authentication

Sign Up as New User

Allows signing up for a new user account. If successful, a new user account will be created and a valid authentication token will be returned.

POST /imusify/api/signup

Parameters

Field Description
fname First name of new user
lname Last name of new user
uname Username of new user
email Email address of new user. Must be valid, unique and unused
password Must be greater than 6 characters
gender Must be m or f ,male or female
mm Month of Birth, Must be an integer
dd Date of Birth, Must be an integer
yy Year of Birth, Must be an integer
password Password
newsletter Optional, Newsletter Subscription
agree Must be on

Response

Field Description
status ok (200) or error (500)
avail_space Available space for Upload
id User account ID
firstname First name of user
lastname Last name of user
email Email address of user
password Hash password of new user
artist Boolean value, artist or user Profile
never_sell Boolean value, Never sell personal music or not
braintreecustId Braintree Customer ID if applicable
loggedin Boolean value, true or false
profileImage Image profile link (avatar)
profileLink Profile link
role_added y or n, Updated roles (e.g. band, music lover …) or not
usertype u or a, User or Artist

Upload partial/full track to the server

Allow the user (artists) to upload chunks of track to the server (The client-side script should split the track into multiple chunks to upload)

POST /imusify/api/uploadfiles/

Parameters

Field Description
name="i" default value, 1
name="r random string value (generated by 3rd party Javascript plugin), e.g. oHidiGhE
name="files[]" name of track (extracted by Javascript script) and content type, e.g. 7 - Bio Book.mp3 and Content-Type: audio/mp3

Response

Field Description
status ok or not found
deleteType Default value: DELETE
deleteUrl URL link to delete chunk(s) or track, e.g. http://beta.imusify.com/imusify/api/uploadfiles/?file=7%20-%20Plymouth%20Suite%20III%20Chanty.mp3
name Name of track, STRING
size Size of all uploaded chunks of track, INTEGER
type Type of audio,e.g. audio/mp3
url Only returned after full track is uploaded, e.g. http://beta.imusify.com/imusify/api/uploadfiles/?file=7%20-%20Plymouth%20Suite%20III%20Chanty.mp3

Complete uploading action to create a new track or song

Allow the user (artists) to upload a new track or song. If successful, the new song is added under Upload page

POST /imusify/save_track

Parameters

Field Description
name="tracktype" Music type, e.g. mp3
name="r random string value (generated by 3rd party Javascript plugin), e.g. oHidiGhE
name="cover_base64_uploaded" URL of uploaded thumbnail, e.g. http://beta.imusify.com/imusify/assets/upload/track/439743deaea69263669f7b18ef3945111373139063.jpeg
name="title" Title of track
name="desc" Description of track
name="mm" Release month, e.g. 03
name="dd" Release date, e.g. 02
name="yy" Release year, e.g. 2017
name="trackuploadtype" Integer value, e.g. 1
name="genre" Integer value, genre ID, e.g. 1
name="trackuploadbpm" Integer value, BPM value of track e.g. 50
name="ispublic" y or n, track is public or private e.g. y
name="sale_available" Integer value, can sell or not, e.g. 1
name="sale_available_ar[]" Integer value, Sell in album or individual track, e.g. 2 (individual track) or 1 (album sale)
name="sell_1" Float value, price for album sale ONLy available if sale_available_ar[] = 1, e.g. 9.99
name="sell_2" Float value, price for single track ONLy available if sale_available_ar[] = 1, e.g. 9.99
name="licence_available" Integer value, the track has license or not, e.g. 1
name="licence_available_ar[]" Integer value, license ID, e.g. 13
name="lic_13" Float value, ONLY available if it has license, Price for license with ID 13, e.g. 35.12

Response

Field Description
status ok or not found
msg Message of uploaded track, e.g. 7 - Chanty Uploaded successfully.
r Random string, STRING
success Status of upload, e.g. success
data Data object of uploaded track (timelength, main_title, title, description, album_song, perLink(unique name of track on server), track_image(full link of thumbnail), track_link (URL of track))
